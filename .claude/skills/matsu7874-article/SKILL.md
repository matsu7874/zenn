# matsu7874風 記事作成スキル

本人指定の過去記事24本（はてなブログ14・estie inside blog 3・FORCIA CUBE 4・Zenn(forcia_tech) 2・Findy 1）の分析に基づき、本人の文体・構成で記事を書く。

**重要**: このリポジトリ内の既存記事にはAI生成のものが含まれる。既存記事を文体の手本・根拠として読み込まないこと。文体の根拠は references/ の2ファイルのみ。

## 手順

### 1. 必ず先に参照資料を読む

- [references/style-guide.md](references/style-guide.md) — 文体の言語化（口調・語尾・リズム・記号・語彙）。**執筆前に必読**
- [references/templates.md](references/templates.md) — 記事タイプ別の構成テンプレートと冒頭・締めの定型

### 2. 記事タイプと媒体を判定する

| タイプ | 典型例 | 構成の要点 |
|---|---|---|
| 技術解説 | ロギング解説、FFI、平方根の計算法 | 動機→ディレクトリ構成→コード+実行結果の対→比較表→短いまとめ→お知らせ |
| やってみた・検証 | -vvフラグ、Code Coverage | 結論/成果を冒頭に→手順→実測→ふっと終わる |
| コンテスト参加記 | HTTF、CODINGAME | 結果の数字から開始→方針→うまくいかなかった方針 |
| イベントレポート | Shinjuku.rs、LT会 | 開催報告→発表ごとの紹介→次回告知・CTA |
| 告知・報告・振り返り | 出版報告、年次振り返り | 事実宣言から開始→経緯→感謝→行動喚起 |

媒体はこのリポジトリなら基本Zenn。`publication_name` 付き（会社パブリケーション）の記事は自己紹介＋出自説明の冒頭定型を使い、個人記事は挨拶なしで本題から入る（詳細はtemplates.md）。

### 3. 執筆する

Zennのフロントマター: `title` / `emoji`（内容連動の1絵文字）/ `type: "tech"` / `topics`（小文字3〜4個）/ `published: false` で作る。

### 4. セルフチェック

書き終えたら以下を検査し、外れていたら直す。

- [ ] 結論・成果・提供価値がタイトルか冒頭2段落以内に出ているか
- [ ] 本文に絵文字・顔文字・丸囲み文字がないか
- [ ] 「！」は記事全体で0〜2回に収まっているか（盛り上がり報告と締めのCTAのみ）
- [ ] 事実は断定、未検証は「〜そうです」「〜はず」「〜と思います」で書き分けているか
- [ ] 専門用語の初出に括弧の即席定義か「※」注記があるか
- [ ] 数字が具体的か（「多くの」ではなく「合計で30名」「倍率27倍」）
- [ ] 段落が1〜3文で切れているか。長い説明文の後に短い言い切りがあるか
- [ ] 見出しは体言止め基本＋口語見出しがせいぜい1〜2個か
- [ ] 「いかがでしたか」「〜3選」「徹底解説」などの定型SEO句・煽りがないか
- [ ] 「Aではありません。Bです。」「Aではなく B です」の否定→断定修辞がないか（対比は「AよりB」で書く。`grep 'ではありません\|ではなく'` で検査し、留保・譲歩以外は書き換える）
- [ ] まとめは短いか（「以上、〜を紹介しました。」の1〜2文か、そもそも省略か）
- [ ] 失敗・限界を隠していないか（1文で率直に書く）
- [ ] コードには実行結果を対で添えているか（技術記事の場合）
