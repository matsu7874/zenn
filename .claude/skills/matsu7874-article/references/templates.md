# 記事タイプ別テンプレート

冒頭・構成・締めの定型。例文はすべて本人指定の過去記事（外部24本）からの逐語引用。リポジトリ内の既存記事はAI生成を含むため根拠にしない。

## 媒体別の冒頭定型

### Zenn（会社パブリケーション: forcia_tech / estie）

「（挨拶＋）肩書＋名前＋出自説明＋記事の目的宣言」。

> これは、FORCIA Advent Calendar 2021の1日目の記事です。エンジニアの松本(@matsu7874)です。

> こんにちは、エンジニアの松本です。この記事は Shinjuku.rs #14 において『Rustでロギングってどうすればいいんですか？』というタイトルでデモを中心に発表した内容をブログ記事としてまとめ直したものです。

続けて記事の目的を1文で宣言:

> この記事ではFFI(foreign function interface)を使って既に書かれたプログラムを活用しながら、一部をRustに置き換えていく方法について解説します。

### estie inside blog / 会社ブログ

「肩書＋名前(@matsu7874)＋『〜の傍ら』の課外活動＋記事の目的宣言」の3ステップ。

> SWEの松本([@matsu7874](https://twitter.com/matsu7874))です。日頃プロダクト開発に邁進する傍ら、「月末LT会」という社内イベントを開催しています。

> エンジニアの松本です。（FORCIA CUBEでは「技術本部エンジニアの松本です。」「新卒4年目、プロダクト部の松本です。」のように所属・年次を添える）

背景説明のあと「さて、」で本題に転換する:

> さて、5/21(火)に4回目となるRustのLT会 "Shinjuku.rs #4" を開催しました。

### はてなブログ / Zenn個人記事

挨拶なしで事実・結果・成果物から入る。

> 生成AIを使ってメールからこんな感じでカレンダーに登録されるようにしました。（＋スクリーンショット）

> 1,297,045点でオープン7位でした。（点数を得た参加者は83人）。

雑学フック→問いかけ→目的宣言の教育型導入も使う:

> √5 = 2.2360679…であることが知られていて『富士山麓オウム鳴く』のごろ合わせも有名です。（…）ところで平方根の値を計算する効率の良い方法を知っていますか？ この記事では平方根の値を求める方法として、計算方法を3つ紹介します。

Tips記事では挨拶より先に「結論」セクションを置いてよい:

> 結論: build.rs で print デバッグができなくて困っている方は `cargo build -vv` を実行すると幸せになれると思います。

## タイプ1: 技術解説記事

```
（冒頭定型: 自己紹介＋記事の出自＋目的宣言）
# 動機付けセクション（なぜそれが大切か: 「ロギングは大切」）
# ディレクトリ構成・準備（ツリーで提示してから解説に入る）
# パターン別／選択肢別にH2-H3で分割
  見出しは「パターンA: C言語やPythonからRustを使う」「環境変数に応じて出力するログレベルを変更するenv_logger」形式
  各節: 説明→コード（ファイル名明示）→実行結果→短評、を繰り返す
  複数言語で実装を並記することもある（Python＋Rust）
# 比較表（メリット・デメリット、実測値）
# まとめ（1〜2文: 「本記事では…方法について解説し、より詳しい資料へのリンクを提供しました。」
  または使い分け指針: 「大きな桁数が必要であれば開平法を、そうでなければ高速なニュートン法を使うと良さそうです。」）
# お知らせ（採用・イベント告知）→「この記事を書いた人」（媒体による）
```

## タイプ2: やってみた・検証記事

```
結論 or 成果物を冒頭に（タイトル自体が結論でもよい）
動機（事実→不便→期待の3段: 「〜が多くあります。」「〜も面倒になります。」「〜できれば気持ちよく生活することができそうです。」）
# 環境構築 / 手順（コマンドは「※先頭の $ から入力が必要です。」）
# やってみた / 実測（表・スクリーンショット・実行ログで裏付け）
# 分かったこと・分からなかったこと（未検証は「〜そうです」で正直に）
まとめは省略可。「ちなみに〜」の補足でふっと終わってよい
```

## タイプ3: コンテスト参加記

```
結果の数字から開始: 「1,297,045点でオープン7位でした。（点数を得た参加者は83人）。」
# 問題概要
# 方針
# 実装（「特に面白い実装はありませんが、各関数が更新する変数は少ないので読みやすいはず。」）
# 試したがうまくいかなかった方針 ← 失敗に独立セクション。1項目1〜2文で切り捨てる
  「直前の点と大きく向きが違う場合に無視する。手損でした。」
# 感想（「第一に楽しかったです。」から。次回への課題＋「対戦ありがとうございました。」）
```

## タイプ4: イベントレポート

```
（所属）の松本です。＋定例イベントの背景1〜2文
「さて、」で開催報告（日付・参加人数の数字入り）
  「さて、5/21(火)に4回目となるRustのLT会 "Shinjuku.rs #4" を開催しました。」
# 発表ごとにH3（「発表タイトル | 発表者さん」形式）
  内容要約＋印象（「〜とのこと。」の伝聞名詞止め）＋会場の反応（「テストがオールグリーンになるたびに会場からは拍手が。」）
# 懇親会・表彰の様子（参加者の声をかぎ括弧列挙: 「メンターのレスが早い」「飯がうまい」）
# 次回告知・メンバー募集 → 採用導線で締める
  「皆様からのLTをお待ちしています。」「ご応募をお待ちしております！」
```

## タイプ5: 告知・報告・振り返り

```
事実の宣言から: 「先日、Rustについての書籍を共著で出版させていただきました。」
経緯・詳細（数字と固有名詞を正確に: 担当章、刷数、応募者数）
感謝（具体的な相手名: 「中でも初田さんと吉川さんには頭が上がりません。」）
締めは行動喚起: 「ぜひカスタマーレビューを書いていただけませんでしょうか。」「興味のある方はチャンスと思ってご応募ください。」
```

年次振り返りのみ**常体**で書く:

```
「2020年はCOVID-19の世界的流行があった。」
トピック別H2（仕事/LT/出版/競プロ/学習/…）
# まとめ（2文で総括）
# おまけ：遊んだゲーム（『名前』＋一言短評: 「『Oxygen Not Included』汚染の管理が難しい、未クリア。」）
```
